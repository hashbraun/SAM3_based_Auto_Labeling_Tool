# 품질관리 보고서

## 검토 요약
- 검토 파일 수: 11개
- 발견 이슈: 6건 (심각 3 / 경고 2 / 참고 1)
- 수정 완료: 6건
- 미수정 (수동 필요): 0건

## 발견 및 수정 내역

### [심각] label.py — POST /label/batch 경로가 POST /label/{image_id}에 가려짐

- **파일**: `backend/routers/label.py`
- **문제**: FastAPI는 라우터를 등록 순서대로 매칭한다. 기존 코드는 `POST /label/{image_id}`가 먼저 등록되어 있어, `POST /label/batch` 요청이 `image_id="batch"`로 처리되었다. 그 결과 배치 라벨링 API가 실질적으로 호출 불가능했다.
- **수정**: `POST /label/batch`를 `POST /label/{image_id}` 앞으로 이동하여 정확한 경로가 우선 매칭되도록 변경.
- **근거**: FastAPI(Starlette) 라우터는 등록 순서가 곧 우선순위이므로, 고정 경로 세그먼트(`batch`)가 경로 파라미터(`{image_id}`)보다 반드시 먼저 등록되어야 한다.

---

### [심각] export.py — GET /export/zip 경로가 POST /export/{image_id}에 가려짐

- **파일**: `backend/routers/export.py`
- **문제**: `POST /export/{image_id}`가 먼저 등록되어 있었다. HTTP 메서드가 달라 직접 충돌은 아니지만, FastAPI의 경로 파라미터 매칭 우선 순위 상 `GET /export/zip`은 `{image_id}="zip"`으로 오인될 가능성이 있고, 미래 메서드 추가 시 실제 버그로 이어진다. 또한 `import os`가 사용처 없이 포함되어 있었다.
- **수정**: `GET /export/zip`을 `POST /export/{image_id}` 앞으로 이동하고, 미사용 `import os`를 제거.
- **근거**: 고정 경로를 파라미터 경로보다 먼저 등록하는 것이 FastAPI 표준 관행이다.

---

### [심각] label.py — CorrectionState 미사용 import

- **파일**: `backend/routers/label.py`
- **문제**: `from state import CorrectionState`가 있으나 `label.py` 내에서 `CorrectionState`를 전혀 사용하지 않는다. 실제 사용 위치는 `correct.py`이다.
- **수정**: `label.py`에서 `from state import CorrectionState` 라인 제거.
- **근거**: 미사용 import는 코드 가독성을 해치고, 린터(ruff) 경고를 발생시킨다.

---

### [경고] sam_service.py — mask_input 존재 시 multimask_output=True 오류

- **파일**: `backend/services/sam_service.py`
- **문제**: SAM `predict_points`에서 `mask_input(prev_logits)`이 있는 경우에도 `multimask_output=True`를 고정으로 전달했다. SAM 공식 코드에 따르면 `mask_input`을 제공할 때는 `multimask_output=False`이어야 하며, `True`로 설정하면 내부적으로 예외가 발생하거나 mask_input이 무시된다.
- **수정**: `multimask_output=prev_logits is None`으로 변경. `prev_logits`가 없을 때(초기 호출)는 `True`, 있을 때(후속 보정)는 `False`가 전달된다.
- **근거**: `segment_anything/predictor.py` 소스에서 `mask_input`과 `multimask_output=True`는 함께 사용할 수 없다고 명시되어 있다.

---

### [경고] correct.py — reset_correction에서 det_idx 범위 검사 누락

- **파일**: `backend/routers/correct.py`
- **문제**: `add_point`에는 `det_idx >= len(img.seg_results)` 검사가 있지만 `reset_correction`에는 없었다. 범위 밖의 `det_idx`로 요청 시 `img.seg_results[det_idx]`에서 `IndexError`가 발생하고 500 에러가 반환된다.
- **수정**: `img.corrections.pop` 전에 `add_point`와 동일한 범위 검사를 추가.
- **근거**: 동일한 경로 파라미터를 받는 두 핸들러가 서로 다른 수준의 입력 검증을 갖는 것은 불일치이며, 예측 불가능한 서버 오류의 원인이 된다.

---

### [참고] LabelCanvas.tsx — useEffect deps 배열 검토

- **파일**: `frontend/src/components/LabelCanvas.tsx`
- **문제**: `toCanvas` 함수가 컴포넌트 렌더마다 새로 생성되고 `useEffect` 내부에서 사용되나 deps 배열에 없다. 그러나 `toCanvas`의 계산에 사용되는 모든 값(`offsetX`, `offsetY`, `drawW`, `drawH`)이 이미 deps 배열에 포함되어 있으므로, 실질적인 stale closure 문제는 발생하지 않는다.
- **수정**: 수정 없음. `toCanvas`를 deps에 추가하면 렌더마다 새 함수 참조가 생성되어 무한 리렌더 루프가 발생한다. 현재 구조가 올바르다.
- **근거**: `toCanvas`의 모든 상위 의존 값이 이미 deps에 포함된 derived 계산이므로 추가 조치 불필요.

## 구조 개선 제안

- `routers/export.py`: `export_image`와 `export_all_zip`의 로직(build_yolo_lines 호출)이 중복된다. 향후 두 함수가 복잡해지면 공통 헬퍼로 추출을 고려.
- `sam_service.py`: `predict_points`의 반환값이 `multimask_output` 값에 따라 배열 크기가 달라진다(`True`이면 3개, `False`이면 1개). `correct.py`의 `int(np.argmax(scores))`는 단일 마스크(False)일 때도 동작하지만, 명시적 인덱스 `[0]`으로 바꾸면 의도가 더 명확해진다.

## 잔여 리스크

- `LabelCanvas.tsx`의 `toCanvas` 함수는 현재 `useEffect` deps 밖에 있다. ESLint `exhaustive-deps` 규칙을 사용하는 환경이면 경고가 발생할 수 있다. `useCallback`으로 메모이제이션하거나 `useEffect` 내부로 이동하는 것을 장기적으로 고려.
- SAM / DINO 모델 경로가 `os.environ`으로만 설정된다. 파일이 존재하지 않을 경우 startup 시 명확한 에러 메시지 없이 실패할 수 있다. 경로 유효성 검사 추가를 권장.
